DANIEL HAIM

RISC-V CPU Design

As a part of my computer architecture class at Berkeley, I collaborated with Amber-Mae Skutelsky in designing a RISC-V CPU using Logisim. We built all of the sub-components of the CPU including the control unit, branch comparator and, immediate generator. We used Read-only memory (ROM) to convert the op-codes for each incoming instruction into a set of bits, each corresponding to a control unit function. We then connected all of the different sub-components to build a functioning two-stage CPU. The first stage took care of the Instruction fetch (IF) while the second stage handled the rest of the stages including the Instruction decode (ID) execute using the ALU (EX) memory access (MEM) and write back (WB). This CPU is capable of avoiding pipeline hazards such as read after write by for example inserting nop instructions to ensure the correct execution to consecutive add instruction where the second instruction depends on the result of the first
